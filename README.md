# Product Catalog Application - API & Routing Documentation

This application uses Spring Web MVC to serve HTML templates rather than a traditional JSON REST API. Below are the documented routes, their parameters, and the expected behaviors.

## Models

### `Product`
The core entity managed by this application.

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `Long` | Unique identifier (Auto-generated) |
| `name` | `String` | The name of the product |
| `price` | `Double` | The price of the product |
| `category`| `String` | The category the product belongs to |

---

## Web Routes / Endpoints

### 1. Home Page
- **URL Path**: `/`
- **HTTP Method**: `GET`
- **Description**: Serves the main landing page of the application.
- **Returns**: `index.html` Thymeleaf template.

### 2. Show Add Product Form
- **URL Path**: `/add`
- **HTTP Method**: `GET`
- **Description**: Displays the form required to add a new product to the catalog.
- **Returns**: `add-product.html` Thymeleaf template. It injects a new, empty `Product` object into the model for the form to bind to.

### 3. Submit New Product
- **URL Path**: `/add`
- **HTTP Method**: `POST`
- **Description**: Processes the submission from the "Add Product" form. Saves the new product into the database.
- **Request Parameters (Form Data)**:
  - `name` (String)
  - `price` (Double)
  - `category` (String)
- **Returns**: HTTP 302 Redirect to the `/display` endpoint.

### 4. Display All Products
- **URL Path**: `/display`
- **HTTP Method**: `GET`
- **Description**: Retrieves all products from the database and displays them in a catalog view.
- **Returns**: `display-products.html` Thymeleaf template, populated with a `products` list attribute containing all saved `Product` objects.
